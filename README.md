# .github

*Community health files for the @GitHub organization*

## Overview

This repository contains default community health files that apply to all public repositories in the @GitHub organization that don't have their own versions of these files. These files help establish community standards and provide consistent guidance across all GitHub projects.

## What's Included

This repository contains the following community health files:

- **[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)** - Our Code of Conduct based on the Contributor Covenant, outlining expected behavior and enforcement policies for our community
- **[CONTRIBUTING.md](CONTRIBUTING.md)** - Guidelines for contributing to GitHub projects, including how to submit pull requests and follow our development workflow
- **[SECURITY.md](SECURITY.md)** - Information about our security vulnerability disclosure process and Bug Bounty program
- **[profile/README.md](profile/README.md)** - The public-facing README for the GitHub organization profile page

## How It Works

When a repository in the @GitHub organization doesn't have its own community health file (like CODE_OF_CONDUCT.md or CONTRIBUTING.md), GitHub automatically uses the version from this `.github` repository. This ensures consistent community standards across all our projects without requiring duplication.

### For Repository Maintainers

If you maintain a repository in the @GitHub organization:

- These files apply by default to your repository
- You can override any file by adding your own version to your repository
- Custom versions should follow the same spirit and standards as these defaults

### For Contributors

If you're contributing to a GitHub repository:

- Review the CODE_OF_CONDUCT.md to understand our community standards
- Read CONTRIBUTING.md for guidelines on submitting contributions
- Check SECURITY.md if you discover a security vulnerability
